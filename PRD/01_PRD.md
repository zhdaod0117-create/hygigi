# Retriever Nest 2.0 (주말커플 확장) — PRD

> 생성일: 2026-07-07
> 생성 도구: Show Me The PRD
> 대상: 기존 Retriever Nest(커플 공유 투두/메모, Next.js 14 + Supabase)에 추가하는 기능 확장

---

## 1. 제품 개요

### 한 줄 요약
평일에 못 만나는 주말커플이 각자 편한 시간에 하루와 감정을 남기고, 쌓인 기록을 함께 돌아보는 "비동기 소통 공간"을 기존 앱에 추가한다.

### 해결하는 문제
- **대화 휘발**: 카톡으로 나눈 좋은 말·소중한 순간이 스크롤 속에 묻혀 다시 찾을 수 없다.
- **타이밍 불일치**: 내가 힘든 순간에 상대는 회의 중. 실시간 대화는 둘 다 직장인인 평일에 자주 어긋난다.

### 핵심 가치
- 카톡 대체가 아니라 **카톡이 못 하는 것**만 한다: 흘러가지 않고 **쌓이는** 기록, 답장 부담 없는 **신호**.
- 기존 앱의 픽셀 리트리버가 감정 체크인·일기 작성에 반응해서 "기록하는 행위" 자체를 귀엽고 가볍게 만든다.
- 이미 쓰고 있는 앱(투두/메모)에 얹으므로 새 앱 설치·가입 장벽이 없다.

## 2. 사용자

### 주요 사용자
- **누구**: 둘 다 직장인인 주말커플 (금·토·일만 만남, 평일은 카톡)
- **상황**: 출근길 지하철, 회의 사이 3초, 야근 중 잠깐, 자기 전 침대
- **목표**: 바쁜 평일에도 서로의 상태를 놓치지 않고, 주말에 만났을 때 "그날 그거 어땠어?"로 이어지는 대화 재료를 쌓기

### 사용자 시나리오
1. **아침**: 출근한 A가 헤더의 날씨 버튼에서 ☁️(흐림)을 탭 → B의 화면에서 강아지가 "A의 마음이 흐려요.." 말풍선
2. **점심**: B가 회의 끝나고 확인 → ☁️에 🥺 리액션 한 번 (답장 부담 없음)
3. **저녁**: 앱이 띄운 오늘의 질문 "오늘 하루 중 제일 뿌듯했던 순간은?"에 A가 답 작성 → B 화면엔 "A가 오늘의 일기를 썼어요" 만 표시 (내용은 잠김)
4. **밤**: B도 답 작성 → 그 순간 서로의 답이 공개, 강아지가 "둘 다 썼다! 멍!" 
5. **주말**: 만나서 지난주 일기·감정 기록을 함께 넘겨보며 대화

## 4. 사용자 흐름 (User Flow)

### 화면 구조 (기존 대시보드 확장)
```
하단 탭 3개 (모바일 우선):
🏠 홈      — 기존 할일/메모 + 상단 "오늘의 우리" 스트립 (서로의 감정 날씨 + 출퇴근 상태)
📖 일기    — 오늘의 질문 + 내 답 작성 + (공개 시) 상대 답 + 리액션
📚 기록    — 날짜별 타임라인 (감정 로그 + 지난 일기 아카이브)
```

### 핵심 흐름: 교환일기
```
일기 탭 진입 → 오늘의 질문 표시 → 내 답 작성(당일 수정 가능)
  → 상대가 아직이면: "상대방이 쓰면 서로 공개돼요" + 상대 답 잠금 카드
  → 둘 다 작성 완료: 서로의 답 공개 + 강아지 축하 + 읽음/리액션
```

### 핵심 흐름: 감정 체크인
```
홈 헤더의 내 날씨 버튼 탭 → 5종 날씨 시트 → 선택 (+ 선택적 한 줄)
  → 저장 즉시 상대 화면 실시간 반영 (기존 Supabase Realtime 재사용)
  → 상대는 탭 한 번으로 리액션
```

> ⚠️ 가정: 일기는 **당일(KST)만 작성 가능**하고 지난 날짜는 빈 채로 남습니다 (놓친 날을 소급 작성하면 "둘 다 써야 공개" 규칙이 복잡해지므로). — 아니라면 알려주세요

## 5. 성공 기준

- [ ] 평일(월~목) 중 3일 이상 두 사람 모두 감정 체크인 or 일기 작성 (주간 기준)
- [ ] 교환일기 "둘 다 작성 → 공개" 전환율 60% 이상 (한쪽만 쓰고 끝나지 않음)
- [ ] 감정 체크인 소요 시간 3초 이내 (탭 2번)
- [ ] 기존 기능(투두/메모/실시간 동기화) 회귀 없음

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 이번 확장에서 만들지 않습니다.

- **채팅 기능** — 이유: 카톡을 대체하지 않는다. 실시간 대화는 카톡의 영역.
- **AI 감정 분석/조언** — 이유: 둘만의 기록에 제3자 해석은 부담. 데이터만 잘 쌓으면 충분.
- **공개 SNS/친구 기능** — 이유: 2인 폐쇄 공간이 핵심 가치.
- **네이티브 앱 출시** — 이유: 기존 PWA 유지. 웹푸시(P3)로 알림 욕구 우선 검증.
- **월경/건강 트래킹** — 이유: 민감 데이터, 별도 검토 필요.
